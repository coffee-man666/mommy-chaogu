## 安全最佳实践

部署此项目时：

1. **不要在代码中硬编码密钥**
   - `SERVER_CHAN_KEY` 必须用环境变量
   - 数据库连接字符串用 env var

2. **限制网络暴露**
   - 默认 `mommy-web --host 127.0.0.1`（仅本机）
   - 如需远程访问，用反向代理 + HTTPS（nginx / caddy）

3. **数据备份**
   - `data/watchlist.db` 包含自选股 + 持仓
   - 建议定期 `cp` 到外部存储

4. **依赖更新**
   ```bash
   uv lock --upgrade
   uv sync
   ```

5. **Server酱 推送**
   - 免费版 5 条/天限额，避免被刷爆
   - VIP key (`SCT` 开头的) 是付费资源，妥善保管

## 已知风险

- **Web 端点无认证** — 设计为内网使用，公网暴露需自加认证层
- **依赖链攻击** — 第三方库 (efinance / pandas) 依赖东财 / 腾讯服务器，可能有上游漏洞
- **数据准确性** — 行情数据依赖第三方源，不保证 100% 准确（投资有风险）
