## 一、TL;DR — 3 条铁律

1. **业绩预告 = 事件驱动跳空窗口**。券商预测只是锚点，**实际披露值 vs 预测**才是真信号。
2. **5 日资金流是噪声，半年报周期才是 alpha**。在 7/15-8/31 这 45 天，主线资金的进出场决定一切。
3. **Convexity > Alpha 本身**。预测增长 +200% 以上的标的具备期权式 payoff，**赌超预期的赔率远高于赌精确数字**。

## 三、监控规则（T-7 / T-0 / T+1 节奏）

### T-7 天（披露日前 7 天）

| 触发条件 | 信号 | 操作 |
|---|---|---|
| predicted_high > +100% AND 当前仓位 < 5% | 🟡 关注 | 加入观察池 |
| predicted_high > +100% AND 持仓 ≥ 5% | 🟢 强催化 | 评估是否加仓 |
| predicted_high < 0%（下滑股）| 🔴 风险 | 财报前 5 天减仓 |

**核心理由**：高弹性股在 T-7 时容易出现"聪明钱建仓"动作（先于公告的资金流入）。

### T-0 天（披露日）

**业绩预告形式**（粗范围）：
```
"预计净利润同比增长 +150%~+200%"
"预计净利润同比增长 +50%~+80%"
```

**业绩快报形式**（精确数字）：
```
"净利润 12.5 亿元，同比增长 +175%"
```

**比对规则**：
```
actual_value = 快报数字 或 预告中位数
predicted_range = (predicted_low, predicted_high)
predicted_mid = (low + high) / 2

if actual_value > predicted_high:
    → 🟢 SUPER_BEAT  （超预期，最强信号）
elif actual_value >= predicted_low:
    → 🟡 MEET  （符合预期，中性）
else:
    → 🔴 MISS  （低于预期，最弱信号）
```

### T+1 天（披露日次日）

| 信号 | 操作 | 仓位调整 |
|---|---|---|
| 🟢 SUPER_BEAT | BUY / 加仓 | +50% 仓位 |
| 🟡 MEET | HOLD | 不动 |
| 🔴 MISS | SELL / 减仓 | -50% 仓位 |

**关键**：T+1 开盘常出现跳空，**信号 → 行动要在 9:30-9:35 完成**，否则容易追高/杀跌。

## 四、4 大实战策略

### 策略 1：超预期交易（核心策略）

**适用**：所有 41 家业绩前瞻股

**公式**：
```
收益 = P(超预期) × 跳空幅度 × 仓位
     ≈ 0.4 × (+5%~+15%) × 仓位
```

**实操**：
1. T-7 入选 → T-0 比对 → T+1 行动
2. 单股仓位 ≤ 10%
3. 止损：跌破 T-7 期间最低价

**为什么有效**：A 股业绩预告日 70% 跳空开盘，方向高度可预测。

## 六、风险控制

### 止损规则（铁律）

1. **单股止损**：跌破 T-7 期间最低价 → -50% 仓位
2. **单股最大亏损**：单笔交易最大亏损 ≤ 投入本金 × 5%
3. **板块止损**：板块整体回撤 > 15% → 减仓 30%

### 黑天鹅应对

| 事件 | 应对 |
|---|---|
| 业绩"变脸"（预告后修正）| T+1 一律先减仓 50% |
| 行业政策突变 | 24h 内评估，必要时清仓 |
| 大盘系统性风险（>5% 跌幅）| 全线减仓 30%，等反弹 |

### 心理纪律

- **不预测，只反应**：超预期/低于预期是事后才知道，别预测"会超"
- **不在披露日交易**：T-0 不动，等 T+1 跳空再说
- **不重仓单股**：再看好也别超 10%

## 七、4 条交易信号定义

> 待实现：未来会作为 Rule 加入 `signals/` 模块

### Rule 1: earnings_beat
```python
if actual_value > predicted_high:
    signal_type = "earnings_beat"
    severity = "high" if (predicted_high - predicted_low) > 100 else "medium"
    action = "BUY"
```

### Rule 2: earnings_meet
```python
if predicted_low <= actual_value <= predicted_high:
    signal_type = "earnings_meet"
    severity = "low"
    action = "HOLD"
```

### Rule Rule 3: earnings_miss
```python
if actual_value < predicted_low:
    signal_type = "earnings_miss"
    severity = "high" if (predicted_mid - actual_value) > 50 else "medium"
    action = "SELL"
```

### Rule 4: earnings_approaching
```python
days_to_disclosure = disclosure_date - today
if days_to_disclosure <= 7 and predicted_high >= 100:
    signal_type = "earnings_approaching"
    severity = "medium" if predicted_high >= 200 else "low"
    action = "ALERT"
```

      → 操作：HOLD

### 实操要点

- **T-7（7/13）评估**：当前仓位 + 业绩弹性 → 决定是否加仓

## 十一、决策心法（实战总结）

> 这些是从 7/1 实战 + 历史 MEMORY 沉淀的认知

1. **绝对资金值不可信，要用 ratio (bp = 1/10000)**
   - 同样 5,000 万净流入：茅台 vs 凯美特气差 1000 倍
2. **5 日数据是噪声，半年数据是真相**
   - 潍柴 5 日"机构接盘"实为半年 -43 亿派发
3. **量价相关 -0.5 以下是出货信号**
4. **半年报窗口 7/15-8/31 是 alpha 窗口**
   - 业绩催化 > 一切
5. **PE + 现价距 60/250 日均 > +30%**
   - 已 price in，要警惕
6. **超预期交易的关键是 T+1 执行速度**
   - 9:30-9:35 完成，跳空窗口稍纵即逝
7. **Convexity 优先于 Alpha 本身**
   - +200% 以上的标的，赔率远胜 +50% 的标的
