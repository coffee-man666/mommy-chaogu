## 1. CI Pipeline 结构分析 (`.github/workflows/ci.yml`)

### 1.1 当前结构

CI 包含 **2 个 job**：

**`quality` job** (Quality Gates)：
- 触发：`push` 到 `main` / `feature/**`，以及 PR 到 `main`
- matrix: Python `["3.12"]`（单版本）
- steps: `checkout → setup-uv (带 cache) → install Python → uv sync --extra dev → ruff format --check → ruff check → mypy --strict src → pytest -m "not network" -v → step summary`

**`build` job** (Build check)：
- 依赖 `quality` 完成
- 验证 9 个 CLI 入口的 `--help`（watchlist / monitor / cache / earnings / flows / semicon / report / agent / mcp）

### 1.2 优点

- ✅ **完整的 quality gate 链条**：format → lint → type → test，顺序合理
- ✅ **uv cache 加速**：`astral-sh/setup-uv@v3` 带 `enable-cache: true`，减少重复安装
- ✅ **build job 做 CLI smoke test**：验证入口点可加载（比单纯 import 更有价值）
- ✅ **`always()` summary**：即使失败也输出摘要
- ✅ **network 隔离**：`-m "not network"` 确保离线 CI 可靠运行

### 1.3 关键缺口（详见第 3 节）

| 缺口 | 影响 |
|------|------|
| ❌ 无 coverage 检查 / 覆盖率门槛 | 覆盖率可自由下降，当前仅 58% |
| ❌ 无 release/publish job | 无 artifact 构建 |
| ❌ 无安全扫描 | 依赖漏洞无人检测 |
| ❌ 单 Python 版本 | 只测 3.12，但实际 venv 跑的是 3.13.5 |
| ❌ 无 concurrency 控制 | 同 PR 多次 push 会排队跑完整 CI |

### 3.4 测试失败详情（4 个）

所有 4 个失败都在 `tests/test_config.py`，**根因是环境变量污染**：

```
FAILED tests/test_config.py::test_load_config_defaults_when_no_file
  AssertionError: assert 'zai' == 'deepseek'
FAILED tests/test_config.py::test_load_config_reads_toml
FAILED tests/test_config.py::test_env_overrides[DEEPSEEK_API_KEY-...]
FAILED tests/test_config.py::test_create_default_config
```

**根因分析**：当前 shell 环境中残留了 `AGENT_PROVIDER=zai`（由回测任务设置）。测试 `test_load_config_defaults_when_no_file` 假设默认 provider 是 `deepseek`，但 `load_config()` 读取了环境变量覆盖。

**这说明**：
1. 测试**未隔离环境变量**（没有 `monkeypatch.delenv` 清理 `AGENT_PROVIDER`）
2. 在干净 CI 环境（ubuntu-latest）中这些测试**会通过**，但在开发者本地有 env 时会失败
3. 这是 **测试隔离性缺陷**的真实案例

## 5. Lint / Type Check 配置评估

### 5.1 Ruff 配置 (`[tool.ruff]`)

```toml
line-length = 100
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I", "N", "W", "UP", "B", "SIM", "RUF"]
ignore = ["E501", "RUF001", "RUF002", "RUF003"]
```

**评估**：
- ✅ 规则集丰富（9 类）：含 bugbear (B)、简化 (SIM)、pyupgrade (UP)
- ✅ `ignore` 有合理注释（中文 docstring 必然触发 RUF001-003）
- ⚠️ `E501` 被全局忽略（行长超 100 不报），但有 `line-length=100` 作为 format 软约束

### 5.2 Mypy 配置 (`[tool.mypy]`)

```toml
strict = true
python_version = "3.12"
```

**关键问题**：有 **5 个 `ignore_errors = true` override**，显著削弱 strict 的实际作用：

| 模块 | override | 原因 |
|------|----------|------|
| `cache.*` | `ignore_errors = true` | JSON 反序列化类型推导弱 |
| `market_data.tencent_adapter`, `fallback_adapter` | `ignore_errors = true` | Protocol override |
| `web.*`, `agent.*`, `backtest.*` | `ignore_errors = true` | FastAPI/Pydantic 类型弱 |
| `cli` | `ignore_errors = true` | — |

**评估**：`agent`、`web`、`backtest` 是项目最核心、代码量最大的 3 个模块，全部 `ignore_errors` 意味着 `mypy --strict` 实际只覆盖了约 40% 的代码库。strict 标名存实亡。

## 9. 改进建议（按优先级排序）

### P0 — 必须立即修复

1. **修复 `test_config.py` 环境变量隔离**
   - 在 4 个失败测试中加入 `monkeypatch.delenv("AGENT_PROVIDER", raising=False)` 等清理
   - 或用 `monkeypatch.setattr` 强制指定 provider
   - 影响：测试在开发者本地环境也能稳定通过

2. **配置覆盖率门槛**
   - 在 `pyproject.toml` 加 `--cov-fail-under=55`（当前 58%，留缓冲）
   - CI 的 pytest 命令改为 `uv run pytest -m "not network" --cov=mommy_chaogu --cov-fail-under=55`
   - 阻止覆盖率持续下降

### P1 — 短期改进（1-2 周）

3. **添加安全扫描 job**
   ```yaml
   security:
     runs-on: ubuntu-latest
     steps:
       - uses: actions/checkout@v4
       - uses: astral-sh/setup-uv@v3
       - run: uv sync --extra dev
       - run: uv run pip install pip-audit
       - run: uv run pip-audit
   ```
   或配置 Dependabot `.github/dependabot.yml` 监控 `uv.lock`

4. **为 0% 模块补测试**（按风险排序）
   - `flows/service.py` (214 行) —— 资金流是核心功能
   - `semicon/store.py` (153 行) —— 参考库 CRUD，易测
   - `report_render/` (273 行) —— 渲染逻辑易测

5. **修复 Python 版本不一致**
   - CI matrix 改为 `["3.12", "3.13"]`
   - 或锁定开发环境到 3.12

### P2 — 中期改进（1 个月）

6. **添加 CI concurrency 控制**
   ```yaml
   concurrency:
     group: ci-${{ github.ref }}
     cancel-in-progress: true
   ```
   避免 feature 分支重复 push 时浪费 CI 资源

7. **逐步收紧 mypy override**
   - 先去掉 `cli` 的 `ignore_errors`（影响小）
   - 再处理 `backtest/*`（测试充分，类型修复风险低）
   - `agent/*` 和 `web/*` 最后处理（代码量大）

8. **添加至少 1 个端到端集成测试**
   - 建议覆盖："加载自选股 → mock adapter 取行情 → 触发信号 → 推送去重"完整链路
   - 放入 `tests/test_integration/` 目录

9. **为 network 测试添加定期运行的 job**
   - `efinance_adapter.py` 12% 覆盖率是盲区
   - 建议 cron job（如每日）运行 `-m "network"` 测试，失败发通知

### P3 — 长期优化

10. **添加 release/publish job**（构建 wheel + 上传 artifact）
11. **配置 `--cov-report=xml` + 上传到 Codecov**（覆盖率趋势可视化）
12. **考虑 `pre-commit` 钩子**（本地提交前跑 ruff format + check）
