## 一、项目是什么

mommy-chaogu 是一个 A 股投研工具集，从一个"给妈妈用的手机行情工具"起步，逐步长成了涵盖数据采集、信号告警、LLM 分析、记忆系统、回测引擎的完整框架。

它有 11 个 CLI 命令、一个 FastAPI + Vue 3 Web UI、一个 MCP Server，核心代码约 9,000 行 Python，742 条自动化测试。

**它不是一个**：
- 自动交易机器人 — 没有下单接口，不做实盘
- 荐股软件 — 不提供买卖建议，只提供分析工具和数据
- 稳定盈利系统 — 回测 Alpha 仅 +4%（规则引擎），样本外表现未知

**它是一个**：
- 数据采集 + 分析框架 — 多源行情数据、资金流信号、基本面拉取
- LLM agent 平台 — 21 个工具 + 多 provider + 记忆系统
- 回测实验场 — 规则引擎 + LLM 驱动，验证"想法是否靠谱"

## 四、回测引擎：规则能跑赢基准，LLM 还需验证

这是本文最重要的部分。我们有两套回测引擎，用同一份数据跑出了真实结果。

### 4.1 规则引擎：49.3% 命中率，Alpha +4%

规则引擎基于资金流 ratio（主力净流入 / 流通市值，单位 bp）生成信号：

- ratio > +5bp → bullish
- ratio < -5bp → bearish
- |ratio| > 10bp → 强信号

**回测参数**：10 只股票，2026-06-01 ~ 2026-07-03，滑动窗口 5 天，horizon 5 天。

| 指标 | 值 | 说明 |
|------|-----|------|
| 总预测 | 136 | 19 天 × ~7 只/天 |
| 命中 | 67 | 方向 + 幅度匹配 |
| **命中率** | **49.3%** | Wilson 95% CI: [41.0%, 57.6%] |
| Buy-and-hold 基准 | 46% | 同窗口同股票 |
| **Alpha** | **+4%** | 微弱正 Alpha |
| 耗时 | < 1 秒 | 纯计算，无 API 调用 |

**统计显著性**：p=0.93，不能拒绝"策略等同于随机"的零假设。136 条样本不够，需要至少 500+ 条才能得到有统计意义的结论。

#### 但个股分化很有信息量

| 股票 | 命中率 | p-value | 结论 |
|------|--------|---------|------|
| 002594 比亚迪 | **81.0%** | **0.007** | 统计显著有效 |
| 000858 五粮液 | 70.6% | 0.14 | 趋势好但样本不足 |
| 600519 贵州茅台 | 66.7% | 0.39 | 消费股信号偏有效 |
| 688981 中芯国际 | 41.7% | 0.77 | 信号无效 |
| 002129 TCL中环 | 25.0% | 0.15 | 信号弱 |
| 603662 柯力传感 | 20.0% | 0.11 | 信号弱 |
| **002747 埃斯顿** | **11.1%** | **0.04** | **统计显著反向！** |

**关键发现**：比亚迪是唯一统计显著有效的个股（p=0.007）。更有意思的是埃斯顿——11% 的命中率说明资金流信号在这只股票上**系统性失效甚至反向**。如果把埃斯顿的信号反过来用（看多变看空），命中率就变成 89%。

这说明：**资金流信号不是普适的**。大盘消费股（比亚迪、五粮液、茅台）信号有效性高，中小盘制造股（埃斯顿、柯力传感）信号可能误导。

#### 信号强度也重要

| 信号强度 | 命中率 |
|----------|--------|
| 普通（5-10bp） | 46.1% |
| 强信号（10bp+） | **55.3%** |

极端资金流异动（ratio > 10bp）的命中率比普通信号高 9 个百分点。这与直觉一致：大资金异动更可信。

#### 自动知识提炼

规则引擎每次回测后自动生成知识条目，system prompt 从 612 字进化到 2,185 字：

```
[个股] stock:002594：30天回测：资金流信号命中率 81%（21 条验证）（置信度 81%）
[个股] stock:002747：30天回测：资金流信号命中率 11%（9 条验证）（置信度 11%）
[市场] market：2026年6月市场：总命中率 49%，空头或震荡占优（置信度 60%）
```

这些知识会注入到 LLM agent 的 system prompt 中，让后续分析"站在过去经验的基础上"。

### 4.2 LLM 回测：33% 命中率，但高置信度有区分力

用 GLM-4.7 对 3 只中期报股票（兆易创新、北京君正、富创精密）做了 3 天回测：

| 指标 | 值 |
|------|-----|
| 总预测 | 7（1 条过期） |
| **命中率** | **33.3%** |
| Buy-and-hold 基准 | 50% |
| **Alpha** | **-17%** |
| Token 消耗 | 22,575（prompt=4,677, completion=17,898） |
| 成本 | ¥0.0452（约 ¥0.0065/条） |
| 耗时 | ~7.5 分钟（~30s/条） |

33% 的命中率看起来很差，但样本只有 7 条，Wilson CI 宽达 [9.7%, 70%]，什么都说明不了。真正有价值的是内部结构：

| 置信度 | 命中率 | 样本 |
|--------|--------|------|
| 高（≥0.7） | **50%** | 4 条 |
| 中（0.4-0.7） | 0% | 2 条 |

**高置信度预测命中率 50%，中置信度全部失误。** 这意味着 LLM 的自信程度确实与准确率正相关——当它"很确定"时值得关注，当它"不太确定"时确实不准。

#### LLM 的独到之处

以 6/30 兆易创新的预测为例：

> bearish, conf=0.80
> 30日跌至815且大单流出ratio-8.58，前日冲高840主力仅流入+1.02，诱多嫌疑。

LLM 识别出了"价格涨但资金不跟进"的量价背离——这种模式化的判断是规则引擎做不到的。规则引擎只看 ratio 阈值，LLM 能理解"涨停但资金弱"背后的含义。

#### LLM 的成本瓶颈

GLM-4.7 默认启用深度推理（thinking），每次产生 1,000+ reasoning tokens：

| 指标 | 值 |
|------|-----|
| 有效输出 | ~200 tokens/条（direction + confidence + rationale） |
| 推理消耗 | ~2,500 tokens/条（reasoning） |
| 单次延迟 | 25-30 秒 |
| 每条成本 | ¥0.0065 |

回测 100 条需 50 分钟 + ¥0.65。成本可接受，但速度是规模化瓶颈。

### 4.3 横向对比：谁更适合什么

| 维度 | 规则引擎 | LLM (GLM-4.7) |
|------|----------|---------------|
| 命中率 | **49.3%** | 33.3%（样本太小） |
| Alpha | **+4%** | -17%（样本太小） |
| 速度 | **秒级 / 136 条** | 30s/条 |
| 成本 | **¥0** | ¥0.0065/条 |
| 分析深度 | ratio + 涨跌幅 | 量价 + 资金流 + 趋势 + 模式识别 |
| 可解释性 | 规则透明 | 理由含上下文 |
| 最佳场景 | **大规模筛选** | **精选个股深度分析** |
| 最差场景 | 信号反向的股票 | 需要速度的批量任务 |

**结论**：规则引擎是扫雷达，LLM 是放大镜。先用规则引擎筛选出极端信号（ratio >10bp），再用 LLM 对筛选结果做深度分析，可能是最优组合。

## 五、记忆系统：框架完整，数据在积累

记忆系统有 5 层架构：

| 层 | 用途 | 当前状态 |
|----|------|----------|
| 工作记忆 | 当前对话上下文 | 运行时生效 |
| 情景记忆 | 分析事件流（谁/何时/说了什么） | 回测已写入 27 条事件 |
| 预测验证 | 预测 → 到期验证 → 命中/失误 | 回测已写入 136 条预测 |
| 语义知识 | 从经验中提炼规律 | 规则引擎自动生成 10 条 |
| 向量检索 | 语义相似度搜索 | 可用，依赖 embedding |

记忆系统通过 `MemoryPipeline` 在回测/聊天/报告/监控全入口激活。每次分析后会自动：
1. 写入情景事件（什么时间分析了什么股票，结论是什么）
2. 创建预测记录（方向、置信度、入场价）
3. 到期后自动验证，更新命中/失误统计
4. 从历史验证中提炼规律，注入未来的 system prompt

回测验证了 100% 的可追溯性：每条预测都能追溯到触发它的情景事件（`source_event_id`）。

## 六、诚实的局限

### 统计层面

- **样本太小**：规则引擎 136 条、LLM 7 条，都远不足以得出统计显著结论
- **单时间段**：只有 2026 年 6 月一个月的数据，无法评估跨市场环境（牛市/熊市/震荡）的表现
- **过拟合风险**：知识提炼基于回测数据，但 walk-forward 检测尚未在这批数据上运行
- **没有样本外验证**：所有测试都是 in-sample，不知道泛化能力

### 工程层面

- **数据库迁移有遗留**：部分 CLI 命令仍引用旧路径（`data/watchlist.db`），semicon 数据未同步
- **实时模式未经考验**：缓存层全空，从未做过完整的实时行情拉取 → 信号触发 → 微信推送闭环
- **LLM 延迟高**：GLM-4.7 reasoning tokens 导致单次调用 30 秒，不适合实时场景
- **覆盖率有盲区**：核心数据源（efinance adapter）在 CI 中覆盖仅 12%

### 方法论层面

- **只有资金流一个信号源**：没有结合技术指标（MACD/RSI/布林带）、基本面、新闻情绪
- **horizon 固定**：T+3 或 T+5，没有自适应调整
- **无仓位管理**：回测只看方向，不考虑仓位大小、止损止盈、资金分配
- **无滑点/冲击成本模型**：交易成本只算固定费率，不考虑大单冲击

## 七、下一步：哪些值得做

按投入产出比排序：

1. **拉更多数据**（2 周 → 3 个月） — 当前只有 21 天资金流数据，扩到 3 个月可以做有意义的 walk-forward
2. **跨市场环境验证** — 用 regime 分析在牛/熊/震荡不同环境下分别评估信号有效性
3. **规则 + LLM 组合策略** — 规则引擎筛极端信号 → LLM 深度分析 → 人工确认
4. **关闭 reasoning tokens** — GLM-4.7 可关闭 thinking，延迟从 30s 降到 5s，适合批量回测
5. **实时闭环验证** — 跑一周实时行情拉取 → 信号 → 推送，验证工程链路
6. **补 flows/ 测试** — 资金流是核心功能，631 行代码零测试是最大技术债务

## 八、给读者的诚实总结

mommy-chaogu 是一个**工程基础扎实、核心逻辑可运行、但实际效果待验证**的投研框架。

它不是一个能让你赚钱的系统——49% 的规则引擎命中率只比抛硬币好一点点，4% 的 Alpha 在交易成本面前可能蒸发。

但它是一个**值得继续建设的平台**：
- 数据采集层能工作
- 规则引擎秒级出结果，且自动提炼知识
- LLM agent 能做多维度分析，置信度有区分力
- 记忆系统框架完整，正在积累数据
- 回测引擎能评估任何策略的历史表现

如果你对量化投研感兴趣，欢迎拿这些数据做自己的判断。代码开源，回测可复现。
